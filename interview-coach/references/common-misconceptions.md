# 常见误区库（高频考点）

逐追问时主动探这些点——学生答这些题时高频踩的坑。不是要全问，是当你看到回答里出现对应苗头时，往这些方向追问。

---

## HTTP 无状态

**误区：** "无状态 = 不传输数据 / 客户端信息不会交付到服务端"
**真相：** HTTP 能传输任意数据（POST body、headers）。"无状态"说的是**服务端不维持客户端状态记录**，每个请求对服务端都是陌生的。传输和记忆是两件事。
**追问触发词：** 回答里出现"不传输""不会交付""客户端数据传不过去"
**怎么追：** 反问"POST 的 body 算不算传输过去了？" 或 "你登录后第二个请求，服务端凭什么认你？"

---

## Cookie vs Session

**误区：** "Session 比 Cookie 安全所以二选一用 Session" / "Session 不是明文传输所以更安全"
**真相：** Session 依赖 Cookie——sessionId 通过 `Set-Cookie` 下发，客户端用 `Cookie` 头带回。是**依赖关系不是替代关系**。Session 更安全的本质是**敏感数据不落在客户端**，不是"传输加密"（sessionId 照样明文走 HTTP，要靠 HTTPS）。
**追问触发词：** 回答里把 Cookie 和 Session 并列对比、说"Session 替代 Cookie"、说"明文传输"
**怎么追：** 反问"用 Session 的话，sessionId 怎么在客户端和服务端之间传？" 一问就暴露依赖关系。

---

## Session 生命周期

**误区：** "关闭浏览器 → Session 结束（服务端数据销毁）"
**真相：** 关浏览器只是让**客户端的 sessionId（会话 Cookie）丢了**，服务端那份数据还在，要等**超时时间**到了才被销毁。这是两件事，时序不能混。
**追问触发词：** 回答里说"关闭浏览器 Session 结束/销毁/失效"
**怎么追：** 反问"关浏览器后服务端的 Session 数据还在不在？什么时候被销毁？"
