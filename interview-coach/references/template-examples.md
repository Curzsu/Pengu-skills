# 模板生成正反例库

生成标准回答模板时必读本文件。每个知识点配一组 ✅正例 / ❌反例，对照写法。正例符合 SKILL.md 模板规范，反例对应禁止项。

---

## 例1：CORS 跨域（场景化 + 新手友好 + 粗粒度）

### ✅ 正例（配场景 + 发生过程 + 一个场景串三块）

> **展开"什么是跨域"：**
> 同源策略是浏览器的限制，协议/域名/端口任一不一致就是跨域。
> **具体场景：** 前端跑在 `http://localhost:8080`，后端接口跑在 `http://localhost:8081`，你在页面里点按钮想拿后端数据，这个请求就是跨域。
> **发生过程：** 请求照常发出，后端接口也跑了、日志能看到，返回了响应。但响应回到浏览器后，浏览器一查响应头里没有允许 8080 的头，就把响应扣下不给页面，控制台报跨域错误。所以你看到的是"页面报错拿不到数据"，但后端那边其实一切正常。
>
> **展开"CORS 怎么解决"：**
> 后端在响应头里加上 `Access-Control-Allow-Origin`，告诉浏览器"这个来源我允许"。
> **具体场景：** 还是上面那个 localhost:8080 调 8081 的例子。
> **发生过程：** 后端在响应头加上允许 8080 的标记后，浏览器这次检查通过，把响应放行给页面，数据正常拿到。
>
> **主动收尾：**
> 所以跨域是浏览器单方面的限制，后端不拦。要么后端配 CORS 头放行，要么开发时用代理把请求转成同源。

### ❌ 反例1（干讲机制，无场景）

> 跨域是浏览器的同源策略，协议域名端口不一致就是跨域，后端用 CORS 头解决。

**问题：** 没有具体场景和发生过程，纯背定义，面试官一追"你遇到过吗"就露馅。

### ❌ 反例2（多场景发散）

> 跨域场景：a.com 调 b.com、本地开发端口不同、HTTPS 调 HTTP、子域调父域……

**问题：** 每个点换一个场景，发散不连贯，背不下来。应该像正例那样一个 localhost 场景串到底。

## 例2：HTTPS 握手（粗粒度 + 留追问余量）

### ✅ 正例（4步概括 + 1段why，不背协议）

> **展开 HTTPS 握手：**
> 客户端发起握手 → 服务器返回证书客户端验证 → 双方协商出会话密钥 → 对称加密通信。
>
> **主动收尾（为什么混用非对称和对称）：**
> 非对称加密安全但慢，协商密钥时用一次；对称加密快，之后通信用协商出的密钥。各取所长。

### ❌ 反例（细粒度背协议，8步，易被追问穿）

> ClientHello 带随机数1 → ServerHello 带随机数2 和证书 → 验证证书 → 生成预主密钥 → 公钥加密发过去 → 服务器解密 → 三方派生会话密钥 → 对称加密通信

**问题：** 把机制拆成 8 步细节流程 = 背协议。面试官随手挑一步追问（如"预主密钥怎么派生会话密钥"），答不上就露馅。粗粒度 4 步 + 1 段 why 才是讲八股。细节是"追问时能接住"的储备，不是"回答时要说出来"的内容。

## 例3：Redis AOF rewrite（粗粒度 + 场景化）

### ✅ 正例（机制一句话 + 场景 + 发生过程）

> **展开 AOF rewrite：**
> AOF 文件越写越大，Redis 用 fork 子进程把当前内存状态重新写成一份精简的 AOF。
> **具体场景：** 你的 Redis 跑了一阵，AOF 文件涨到几个 G，里面全是重复的增删改命令。
> **发生过程：** 触发 rewrite 后，主进程 fork 一个子进程，子进程按当前内存里的数据生成新 AOF；这期间主进程照常接请求，新写入的命令同时存到旧 AOF 和一个缓冲区；子进程写完后，缓冲区里那部分增量追加到新文件，然后替换旧文件。
>
> **埋钩子（可选，懂才提）：**
> 如果您感兴趣，我可以讲一下 rewrite 期间新写入是怎么不丢的，这块有个缓冲区设计。

### ❌ 反例（细粒度 + 无场景）

> BGREWRITEAOF 触发 → fork 子进程 → 子进程遍历 dict → 写入 temp-rewrite.aof → 主进程写 aof_buf 和 aof_rewrite_buf → 子进程完成发信号 → 父进程 append aof_rewrite_buf → rename 替换原文件

**问题：** 把流程拆成 7 步 = 背源码。正例只讲"fork 子进程重新写 + 期间新写入走缓冲区"，追问能接住"为什么不丢"即可。

## 维护说明

发现新的模板写法坑点时，按"✅正例 / ❌反例 + 问题说明"格式追加。正例必须符合 SKILL.md 模板规范（粗粒度、配场景、留余量），反例必须对应一条禁止项。保持精简，只收"同一种写法坑反复出现"的。
